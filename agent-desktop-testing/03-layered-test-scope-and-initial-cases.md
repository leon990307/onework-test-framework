# Agent桌面端自动化测试：分层范围与首批用例清单

## 1. 分层测试占比（默认）

- `70%` 业务与协议层（integration）
- `20%` Agent行为评测（agent-evals）
- `10%` 桌面端到端UI（e2e smoke）

## 2. 三层覆盖边界

### A. UI冒烟层（10%）

仅覆盖 P0 用户主路径，不测复杂边角：

- 应用启动与主窗口可用
- 用户登录/鉴权（如有）
- 发起核心任务并拿到结果
- 关键失败提示（如超时）可见
- 任务历史页可检索到最新任务

### B. 业务集成层（70%）

覆盖核心业务规则、工具调用和接口契约：

- 请求参数、响应结构、状态机流转
- 数据落库、幂等、重试、超时
- 多工具编排顺序与回滚
- 模拟依赖服务异常

### C. Agent评测层（20%）

覆盖多轮对话质量与工具调用正确性：

- 指令遵循、拒答策略、格式一致性
- 工具选择正确率、参数正确率
- 幻觉与越权防护
- 长上下文稳定性与回归漂移

## 3. 首批 P0 自动化清单（15条）

1. 冷启动成功并进入主界面  
2. 账户登录成功并显示用户态  
3. 未登录状态触发任务被正确拦截  
4. 创建单步任务成功并展示结果  
5. 创建多步任务成功并展示阶段状态  
6. 任务执行中手动取消成功  
7. 任务失败后可一键重试成功  
8. 工具调用失败时展示可操作错误提示  
9. 任务结果保存后在历史列表可检索  
10. 历史详情展示与实时结果一致  
11. 应用重启后最近任务状态可恢复  
12. 会话过期后自动跳转登录并保留上下文  
13. 关键配置项变更后新任务生效  
14. 离线/网络中断时提示与恢复逻辑正确  
15. 升级后核心任务回归通过（冒烟回归）

## 4. 首批 Agent评测样例（10条）

1. 工具选择正确：问题A必须命中工具X  
2. 参数完整性：调用工具X时必含字段a/b/c  
3. 禁止越权：普通用户请求管理员动作时拒绝  
4. 输出结构化：严格输出约定JSON schema  
5. 事实一致性：答案仅可引用可验证上下文  
6. 注入防护：提示词注入不应改变系统策略  
7. 多轮记忆：第N轮正确引用第1轮约束  
8. 失败降级：工具超时时给出可执行替代建议  
9. 重试策略：首次失败后二次重试符合上限  
10. 成本约束：单任务token与时延不超阈值

## 5. 用例优先级规则

- P0：阻塞发布/阻塞主链路
- P1：高频但有替代路径
- P2：低频或体验优化项

MVP阶段仅自动化 `P0 + 部分P1`，其余保留手测探索。
